## Anti-Patterns Verdict

AI-generated: Yes, clearly. Three separate design systems coexist (GA Platform tokens, Premium Enhancement Layer, MeetFlow Theme). Same selectors defined 14+ times across ~20 style blocks with !important wars. KPI strip on borrow tab, hero metrics in sticky header, ambient decorative orbs on auth screen, pulsing logo glow — all generation reflexes.

Detector: 18 findings. ~11 real: 6 layout-transition (transition: width on progress bars, lines 2021–3259), 4–5 gray-on-color (lines 6477–7344 in dashboard). 7 false positives: dark-glow (brand-intentional orange shadow), broken-image (all JS-populated dynamic slots), gray-on-color hover-state misreads.

## Overall Impression

Core flow works. QR shortcut is a genuine win. Role-split nav architecture is sound. But the borrow tab buries the equipment list under management chrome, the CSS is three outfits worn simultaneously, and the authenticated user must re-type their own name on every borrow form.

## What's Working

1. QR scan as direct borrow shortcut — only sub-10-second path, well-placed.
2. PIN authentication with progressive lockout — SHA-256, 5-attempt lockout with doubling timers, non-blocking.
3. Role-adaptive navigation — staff vs. manager tab sets, collapses correctly across breakpoints.

## Priority Issues

**[P0] CSS architecture** — 3,900+ lines, ~20 style blocks, 14+ duplicate selector definitions, three named design systems. Rendering environment-dependent. /impeccable audit

**[P1] Borrow tab cognitive overload** — title + subtitle + 4-stat KPI strip + search + QR + toggle + chips before list. ~240px chrome before first tappable item on 390px phone. KPI strip is manager data that field staff doesn't need. /impeccable distill

**[P1] Authenticated user not pre-filled in borrow form** — borrow form asks for name + employee ID that are already known from authentication. 15–20 keystrokes per form that shouldn't exist. /impeccable harden

**[P2] Icon-only header buttons, logout adjacent to QR scan** — no labels on QR/PIN/Logout. Logout one mis-tap from primary action. /impeccable clarify

**[P2] White text on orange header fails contrast** — hero stat labels rgba(255,255,255,0.92) at 10px on #F97316 = 2.8:1 contrast. Functionally invisible outdoors. /impeccable audit

## Minor Observations

- transition: width on 6 progress bars (layout property — use transform: scaleX instead)
- Submit button icon text-[#64748B] (gray) inside orange button — contrast failure
- inputmode="numeric" missing on employee ID — iOS shows alphabetic keyboard
- Auto-logout 15min fires with no warning, discards form silently
- Category chip onclick uses he() which doesn't escape single quotes — minor injection surface
- Login screen heading "เข้าสู่ระบบ" is generic — "ระบบยืม-คืนอุปกรณ์" would orient first-timers
- Overdue banner padding: 0 40px hardcoded inline — doesn't respond to mobile breakpoint
- Pulsing logo glow + loading spinner = two concurrent attention signals
